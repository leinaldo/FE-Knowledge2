# 浏览器内核的理解
- 主要分成两部分：渲染引擎（Layout Engine 或 Rendering Engine）和 JS 引擎

- 渲染引擎：负责获取网页的内容（HTML、XML、图像等）、解析内容并应用样式信息（例如 CSS），以及计算网页的布局和显示方式，最后将结果输出至显示器或打印机。不同浏览器的内核对于网页语法的解释可能存在差异，因此渲染效果也可能不同。所有网页浏览器、电子邮件客户端以及其他需要呈现网络内容的应用程序都需要内核支持。

- JS 引擎：负责解析和执行 JavaScript 代码，以实现网页的动态交互效果。

- 最初，渲染引擎和 JS 引擎的区分并不十分明确，后来随着 JS 引擎功能日益独立和复杂，现在“浏览器内核”一词通常倾向于特指渲染引擎。

**常见的浏览器内核有哪些**

- Trident 内核：IE 浏览器（现已由 Microsoft Edge 替代），以及早期一些国产浏览器如 360、搜狗等曾采用该内核。[又称 MSHTML，目前该内核已基本停止更新]

- Gecko 内核：Netscape 6 及以上版本，Firefox（FF），Mozilla Suite / SeaMonkey 等。[Firefox 至今仍主要使用该内核]

- Presto 内核：Opera 7 至 12 版本使用。[Opera 浏览器现已转向使用 Blink 内核]

- WebKit 内核：Safari，早期版本的 Chrome 等。[Chrome 及后续的 Edge、Opera 等现代浏览器目前主要采用 Blink 内核（由 WebKit 分支发展而来）]
