## 🔧 怎么开启
```
"use strict"
```
👉 写在：

文件顶部（全局生效）

函数内部（局部生效）

## ⚡ 核心变化（记这几个就够了）

1️⃣ 不允许使用未声明变量
```
x = 1  // ❌ 报错
```
👉 避免隐式全局变量

2️⃣ 一些“以前静默失败”的行为会直接报错

比如：

给只读属性赋值

删除不可删除属性

给基本类型加属性

👉 从“悄悄失败” → “直接报错”

3️⃣ 限制一些危险语法

比如：

禁止重复参数名

禁止八进制写法

禁止 with

👉 提高代码可维护性

4️⃣ this 行为更合理
```
function fn() {
  console.log(this)
}
fn()
```
👉 严格模式下：
```
this === undefined
```
（不是 window）

## 🚀 为什么要用

更安全（避免隐式错误）

更规范（减少奇怪行为）

更利于优化（引擎更好优化）
